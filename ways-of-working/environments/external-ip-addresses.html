External IP addresses (Egress)

! Warning IP address allow lists are not a recommended approach, see further in this article by GDS with guidance on why we try not to do this anymore: A whitelisting approach for Cloud APIs.)

This page lists the external Public IP addresses used by the firewalls in the Azure platform.

Hubs

Within the Azure Platform, the networking of each virtual network will follow a hub / spoke model. Depending on the environment, it will be connected to one of three network hubs:

Hub IP Addresses
Sandbox 20.49.137.196, 20.49.137.7
Non production 20.49.168.141, 20.49.168.17
Production 20.50.108.242, 20.50.109.148

For other environments, you can check which hub the virtual network is connected to by reviewing the virtual network peering. If unsure or any additional queries, please raise a request via the #platops-help channel in Slack.
